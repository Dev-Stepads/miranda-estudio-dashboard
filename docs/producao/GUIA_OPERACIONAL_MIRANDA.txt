1. O QUE O DASHBOARD FAZ
========================================================

O dashboard consolida 3 fontes de dados em tempo real:

  - Nuvemshop (e-commerce): pedidos, clientes, produtos,
    carrinhos abandonados, geografia
  - Conta Azul (loja fisica): NF-e, clientes, produtos,
    geografia por estado
  - Meta Ads (trafego pago): investimento, ROAS, campanhas,
    criativos, alcance

A sincronizacao acontece automaticamente a cada 30 minutos
via GitHub Actions. Ninguem precisa fazer nada manual no
dia a dia.

3. RISCOS CONHECIDOS E COMO EVITAR
========================================================

RISCO 1 — GitHub desativa cron apos 60 dias de inatividade
--------------------------------------------------------------
O que acontece:
  Se ninguem fizer push no repositorio ou acessar o GitHub
  por 60 dias, o GitHub DESLIGA o cron automaticamente.
  O sync para, o banco para de receber dados, e apos 14
  dias o token do Conta Azul expira.

Como evitar:
  Acessar o repositorio no GitHub pelo menos 1x a cada
  2 meses. Pode ser so abrir a pagina:
  https://github.com/Dev-Stepads/miranda-estudio-dashboard

Se ja aconteceu:
  1. Ir em GitHub > Actions > selecionar o workflow
  2. Clicar "Enable workflow"
  3. Se o Conta Azul parou de funcionar, refazer o OAuth
     (pedir ao Matheus/Stepads)

RISCO 2 — Supabase pausa apos 7 dias sem acesso (plano free)
--------------------------------------------------------------
O que acontece:
  O Supabase no plano gratuito pausa o banco se ninguem
  acessar por 7 dias. Com o cron rodando, isso NAO acontece
  porque o sync escreve no banco a cada 30 min.

  Porem, se o cron for desativado (ver Risco 1), o banco
  pode pausar.

Como evitar:
  Manter o cron ativo (ver Risco 1). Ou migrar para o
  plano Pro do Supabase (US$25/mes) que nao pausa.

Se ja aconteceu:
  1. Abrir https://supabase.com e logar com dev@stepads.com.br
  2. Selecionar o projeto miranda-dashboard
  3. Clicar "Restore project"
  4. Aguardar ~2 min e verificar o dashboard

RISCO 3 — Conta Facebook "Angelo Meirelles" perdida
--------------------------------------------------------------
O que acontece:
  A app Marketing "Miranda Dashboard ETL" no Facebook
  esta vinculada a esta conta. Se a conta for bloqueada,
  hackeada ou excluida:
  - O token do Meta Ads CONTINUA funcionando (nao expira)
  - Mas perde-se a capacidade de ADMINISTRAR a app
    (rotacionar token, alterar permissoes)

Como evitar:
  - Manter 2FA ativo na conta
  - Nao compartilhar a senha
  - Adicionar um segundo admin na app quando possivel
    (developers.facebook.com > Miranda Dashboard ETL >
    Settings > Roles)

Se ja aconteceu:
  Criar nova app Marketing, novo System User e novo token.
  Atualizar as variaveis META_SYSTEM_USER_TOKEN e
  META_AD_ACCOUNT_ID no GitHub Secrets e no .env.local.

RISCO 4 — Token do Conta Azul expira (14 dias sem refresh)
--------------------------------------------------------------
O que acontece:
  O refresh_token do Conta Azul e single-use e expira em
  14 dias. O ETL rotaciona automaticamente a cada 30 min.
  Se o cron parar por 14+ dias, o token expira e o Conta
  Azul para de sincronizar.

Como evitar:
  Manter o cron ativo (ver Risco 1).

Se ja aconteceu:
  1. A Miranda precisa re-autorizar a app no Conta Azul
  2. O Matheus/Stepads faz o fluxo OAuth e obtem novo token
  3. Inserir o novo token no Supabase (tabela etl_config)

8. PERGUNTAS FREQUENTES
========================================================

P: Os dados sao em tempo real?
R: Nao. Atualizam a cada 30 minutos. O Meta Ads pode
   ter delay adicional de 15 min (o Meta recalcula metricas).

P: Por que o ROAS do dashboard e diferente do Ads Manager?
R: O dashboard usa atribuicao 7d_click + 1d_view (padrao
   do Meta). Se a conta usar configuracao diferente no Ads
   Manager, os numeros podem divergir ligeiramente.

P: Por que "compras atribuidas" do Meta nao batem com
   vendas da Nuvemshop?
R: Sao fontes diferentes. O Meta conta conversoes via
   Pixel (atribuicao de midia). A Nuvemshop conta pedidos
   reais. Um cliente pode clicar no anuncio e comprar 3
   dias depois — o Meta atribui, a Nuvemshop registra.

P: Posso confiar no faturamento total do dashboard?
R: Sim. Foi validado contra as APIs em 2026-04-13
   (pedido a pedido). A Visao Geral soma apenas vendas
   PAGAS da Nuvemshop + NF-e EMITIDAS do Conta Azul.
   Pedidos pendentes, cancelados e reembolsados NAO
   entram no faturamento.

P: O que acontece se a Miranda mudar de plataforma
   de e-commerce?
R: O dashboard para de receber dados novos da Nuvemshop.
   Historico permanece. Stepads precisaria integrar a
   nova plataforma.
