MODELAGEM DE DADOS — Dashboard Miranda Studio
========================================================
Status: RASCUNHO INICIAL
Última atualização: 2026-04-10

--------------------------------------------------------
PRINCÍPIO GERAL
--------------------------------------------------------
Três fontes → camada canônica → views do dashboard.

Fluxo:
  [Conta Azul] ──┐
  [Nuvemshop]  ──┼──▶ tabelas brutas ──▶ tabelas canônicas ──▶ views
  [Meta Ads]   ──┘

--------------------------------------------------------
TABELAS BRUTAS (staging)
--------------------------------------------------------
raw_contaazul_sales        — raw JSON + colunas extraídas
raw_contaazul_sale_items
raw_contaazul_customers

raw_nuvemshop_orders
raw_nuvemshop_customers
raw_nuvemshop_abandoned_checkouts

raw_meta_insights_campaign
raw_meta_insights_adset
raw_meta_insights_ad
raw_meta_ads_metadata

Regra: tabelas raw são APPEND-ONLY. Nada é atualizado in-place.
Cada coleta tem ingested_at. Deduplicação acontece na camada canônica.

--------------------------------------------------------
TABELAS CANÔNICAS
--------------------------------------------------------

sales (uma linha por pedido)
  sale_id              pk
  source               'conta_azul' | 'nuvemshop'
  source_sale_id       id original na fonte
  sale_date            timestamptz
  gross_revenue        numeric — receita bruta
  net_revenue          numeric — depois de descontos/estornos
  status               'paid' | 'cancelled' | 'refunded' | 'pending'
  customer_id          fk → customers
  payment_method       text
  ingested_at          timestamptz

sale_items (uma linha por item do pedido)
  sale_item_id         pk
  sale_id              fk → sales
  product_id           fk → products
  product_name         text — nome como veio da fonte
  sku                  text — quando disponível
  quantity             int
  unit_price           numeric
  total_price          numeric

products (catálogo consolidado entre fontes)
  product_id           pk (gerado internamente)
  canonical_name       text
  sku                  text unique when not null
  source_refs          jsonb — {conta_azul: id, nuvemshop: id}
  created_at

customers
  customer_id          pk
  source               'conta_azul' | 'nuvemshop'
  source_customer_id
  name                 text
  gender               'M' | 'F' | 'unknown'
  age                  int nullable
  age_range            '18-24' | '25-34' | '35-44' | '45-54' | '55+' | 'unknown'
  state                uf (2 chars) nullable
  city                 text nullable
  inferred_state_from_ddd boolean — rastreabilidade da regra de fallback

abandoned_checkouts (Nuvemshop)
  checkout_id
  created_at
  total_amount
  customer_id          fk nullable

meta_ads_insights (daily)
  date                 date
  level                'campaign' | 'adset' | 'ad'
  campaign_id          text
  campaign_name        text
  adset_id             text nullable
  adset_name           text nullable
  ad_id                text nullable
  ad_name              text nullable
  spend                numeric
  impressions          bigint
  reach                bigint
  clicks               bigint
  purchases            int
  purchase_value       numeric
  -- KPIs derivados (CTR, CPC, CPM, ROAS, ticket) podem vir de views
