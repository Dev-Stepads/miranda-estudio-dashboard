  mes seguinte entravam no mes atual.

    - 0 sales com venc 01/03 saindo (nada pra compensar)

  createSaleFromRecebivel criava sale com status='paid' mesmo quando
  totalPago=0 (recebivel ainda nao pago). Resultado: 194 sales com
  gross=0 poluindo contagem de pedidos.

FIX APLICADO:
  1. sync.ts:
     - processVenda: skip if grossRevenue <= 0 + sale_date = earliestDate + "T03:00:00Z"
     - createSaleFromRecebivel: skip if grossRevenue <= 0 + mesmo fix TZ
  2. Update massa no Supabase:
     - Deletados 194 sales fantasma (gross=0) + seus sale_items
     - Shift +3h em todos 2548 sales CA restantes (via sale_id, paginado)
